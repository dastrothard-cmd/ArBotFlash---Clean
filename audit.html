This page keeps the product honest: what is working, what is broken, and what is next before the eight-kingdom expansion.

Immediate correction made in v0.13.7

The fake “AI Australian — Female” label has been removed from the deck. The speech module now follows the Tree ID Trainer browser speech pattern: speech off/on, voice type, installed voice, test selected voice, reload installed voices, and card-side speech only. It reads names only, not the full profile cards.

Product scope audit

Status after the current clean static build. This is the checklist we work through one lane at a time.

1. Preserve separationArbotFlash remains separate from treeIDflashcards.com, with independent branding and footer.Pass
2. Core 80 dataset80 specimen cards are present with common name, scientific name, family, sourced profile cards and one reviewed image each.Pass / baseline
3. TIDFC voice behaviourBrowser/device speech module now matches the TIDFC control pattern. Server AI speech is not yet wired into this static clean package.Partial
4. TIDFC visual charmCurrent UI is cleaner but still not as polished as TIDFC. Button spacing, card animation and game surfaces need more work.Active
5. Skins/themesForest, Night and Field book skins exist. Original TIDFC theme artwork is not yet integrated pending ownership/licence confirmation.Partial
6. Multi-photo species cardsThe interface has slots for whole form, bark/trunk, leaves/fruit and features, but most taxa still have only one reviewed image.Data pass needed
7. Full info cardsProfile cards are populated from sourced data, but many need richer spreadsheet-style morphology, use, ecology, chemistry and specimen-location fields.Data pass needed
8. GamesStudy and quick quiz exist. TIDFC-style family/genus and binomial/common matching games still need to be ported into this polished shell.Next UI build
9. Progress/accountsLocal progress, accounts, saved decks, synced mastery and friend data are not yet in the clean static shell.Not started
10. DuellingLive/asynchronous duels, leaderboards and achievements remain scoped but not implemented.Not started
11. Offline/PWAThe earlier architecture has offline pack work; the current clean static shell needs a proper service worker and installable behaviour reattached.Reattach later
12. Review/source systemThe v0.12 project has review/source/licensing architecture. The clean public shell currently surfaces only simplified source/licence details.Preserved in main project, simplified in UI
13. Eight-kingdom expansionThe target remains Catalogue-of-Life-scale across the eight kingdoms. Expansion starts only after this card/game framework is solid.Target preserved

1. Finish speech behaviour and button polish. 2. Port TIDFC matching games. 3. Add real multi-photo data per species. 4. Complete profile cards from the spreadsheet-style fields. 5. Reattach PWA/offline. 6. Add accounts/progress. 7. Add duelling. 8. Begin eight-kingdom ingestion.
